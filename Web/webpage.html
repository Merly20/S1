CLIENT/SERVER




In a client-server model, the client—such as a web browser or mobile application—initiates communication by sending a request over a network to a server, which is a remote computer or system specifically designed to process these requests, retrieve the appropriate resources (such as web pages, files, or database information), and return them to the client, enabling the user to interact with online services and applications efficiently and reliably."







COMPONENTS OF WEB APPLICATION




Here’s a very brief list of the main **components** of a web application:
1. **Client (Front-end)** – What users see and interact with (browser, UI).
2. **Server (Back-end)** – Processes requests and runs the app logic.
3. **Database** – Stores and manages data.
4. **API** – Connects front-end and back-end.




TYPES OF WEB CONTENT




"Web content includes various types such as text, images, videos, audio, links, forms, and interactive elements to engage users."




HTTP HTTP REQUEST






An HTTP request is a communication sent from a client, like a web browser, to a server that asks for specific resources such as web pages, images, or data, and it includes information like the request method (GET, POST), headers, and sometimes data payload to instruct the server how to respond."
